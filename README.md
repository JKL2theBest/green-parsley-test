# Тестовое задание: Junior System Analyst

В этом репозитории представлено решение тестового задания для интернет-магазина "Петрушка Зеленая".
Решение структурировано по трем ключевым задачам: анализ, проектирование API и архитектура.

---

## Навигация по решению

### 1. [Анализ требований (Задание 1)](TASK_1.md)
*   Проведен аудит исходного ТЗ, выявлены критические бизнес-противоречия (конфликты лимитов, цен, логики удаления).
*   Разработана исправленная и непротиворечивая версия требований.
*   Составлен список уточняющих вопросов к стейкхолдерам.

### 2. [Проектирование API (Задание 2)](TASK_2.md)
*   Спроектирован метод `GET /api/v1/partners/stores`.
*   Подготовлен JSON-контракт, поддерживающий **Server-Driven UI** (передача цветов и стилей с бэкенда) и динамические слоты доставки.

### 3. [Архитектура PUSH-уведомлений (Задание 3)](TASK_3.md)
*   Разработана архитектурная схема (Event-Driven).
*   Описан поток данных: от события в микросервисе (Producer) через брокер сообщений (Kafka/RabbitMQ) до доставки на устройство (FCM/APNS).
